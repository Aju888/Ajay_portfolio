01 Foundation

DOCTYPE & HTML Tag

Ye website ka pehla aur sabse zaruri hissa hai. Browser yahan se samajhta hai — "ye kya file hai aur kaise padha jaaye?"

HTML index.html — Line 1 to 3
1<!-- YE SABSE PEHLI LINE HOTI HAI -->
2<!DOCTYPE html>

3<!-- Root element — poori website iske andar -->
4<html lang="hi">

5  <head> ... </head>
6  <body> ... </body>

7</html>
<!DOCTYPE html>
Document Type Declaration

Browser ko batata hai: "Ye HTML5 file hai." Agar ye na ho, browser quirks mode mein chala jaata hai — purane rules follow karta hai, layout toot jaata hai.

<html lang="hi">
Root / Parent Element

Puri website iske andar hoti hai. lang="hi" se screen readers (visually impaired ke liye) aur Google samajhta hai ki content Hindi mein hai — SEO ke liye bhi zaroori.

⚠️ Agar DOCTYPE na likha?

Browser "Quirks Mode" mein chala jaata hai. CSS layouts galat dikhenge, fonts size alag honge, aur cross-browser compatibility toot jaayegi. Hamesha LINE 1 par likho.

02 HEAD Section

<head> — Browser Ke Liye Instructions

User ko <head> nahi dikhta, lekin ye bahut zaroori hai. Isme hum browser ko batate hain — fonts kahan se lao, title kya hai, CSS kahan hai, SEO kya hai.

HTML index.html — HEAD Section (poora)
<head>

  /* ── 1. CHARACTER SET ── */
  <meta charset="UTF-8" />
  ↑ Hindi, Emoji, Special chars sahi dikhenge

  /* ── 2. VIEWPORT ── */
  <meta name="viewport"
        content="width=device-width, initial-scale=1.0" />
  ↑ Mobile par zoom-out nahi hoga, sahi size dikhega

  /* ── 3. TITLE ── */
  <title>Rahul Dev | Web Developer Mumbai</title>
  ↑ Browser tab + Google search result mein dikhta hai

  /* ── 4. SEO META ── */
  <meta name="description"
        content="React developer, 3 saal ka experience" />
  ↑ Google search mein title ke neeche jo text aata hai

  /* ── 5. CSS LINK ── */
  <link rel="stylesheet" href="style.css" />
  ↑ External CSS file yahan connect hoti hai

  /* ── 6. GOOGLE FONTS ── */
  <link rel="preconnect" href="https://fonts.googleapis.com"/>
  <link href="https://fonts.googleapis.com/css2?family=Poppins"
        rel="stylesheet"/>
  ↑ Custom fonts Google se load honge

  /* ── 7. FAVICON ── */
  <link rel="icon" href="favicon.ico" />
  ↑ Browser tab par chhota sa icon

  /* ── 8. OPEN GRAPH (Social Share) ── */
  <meta property="og:title" content="Rahul Dev" />
  <meta property="og:image" content="preview.jpg" />
  ↑ WhatsApp/Facebook par share karo to ye image/title aata hai

</head>
🔤

charset="UTF-8"

Hindi text, emojis, ₹ sign — sab sahi dikhenge. Agar na ho, gibberish characters aayenge: "â€" jaisi cheezein.

📱

viewport meta

Ye nahi likha to mobile par website bahut choti dikhai degi — zoom out hokar. 2025 mein ye line mandatory hai.

🔍

title tag

Google is title ko search result mein dikhata hai. 50-60 characters mein keyword daalo — SEO rank improve hoga.

🎨

link rel="stylesheet"

CSS file ko HTML se connect karta hai. Bina is line ke CSS kaam nahi karegi — page unstyled rahega.

<header>
Semantic HTML5 Tag

Sirf ek <div> bhi use kar sakte the, lekin <header> use karne se Google aur screen readers samajhte hain: "Ye page ka header area hai." SEO aur accessibility dono improve hoti hai.

<nav>
Navigation Wrapper

Screen readers is tag ko dhundh ke user ko batate hain: "Yahan navigation links hain." Google bhi is tag ko samajhta hai.

position: sticky
Scroll par chipka rahega

04 Navigation Links

ul / li / a — Nav Links Ki Structure

Navbar ke andar links ka structure ul (unordered list) + li (list item) + a (anchor/link) se banta hai. Ye ek standard pattern hai jo har professional website follow karta hai.

HTML + CSS — Nav Links Detail
HTML — Har Line Ka Reason
<!-- UL: Unordered List — links ka container -->
<ul class="nav-links">

  <!-- LI: List Item — har ek link ka wrapper -->
  <li>
    <!-- A: Anchor tag — actual clickable link -->
    <!-- href="#home" = same page ka section -->
    <a href="#home">Home</a>
  </li>

  <li>
    <!-- href="/about.html" = doosra page -->
    <a href="/about.html">About</a>
  </li>

  <li>
    <!-- Doosri website par jaana -->
    <a href="https://google.com"
       target="_blank"
       ↑ Naya tab mein khulega
       rel="noopener noreferrer">
       ↑ Security ke liye zaruri
      Blog
    </a>
  </li>

  <li>
    <!-- Active class — current page highlight -->
    <a href="#contact"
       class="active">Contact</a>
  </li>

</ul>
CSS — Link Styles

}

  color: #6366f1;

  content: '';
  position: absolute;
  bottom: 0; left: 50%;

AttributeValueKya Karta Hai
href="#home"#section-idSame page ke kisi section par jump karta hai (smooth scroll)
href="/about"/page-nameSame website ka doosra page
href="https://..."full URLDoosri website par jaata hai
target="_blank"_blankLink naye tab mein khulti hai

class="active"custom classCurrent page ka link highlight karne ke liye
🤔 Sirf div use kyon nahi kar sakte?

ul + li use karne ki wajah: Screen readers tab-by-tab links ko navigate karte hain — keyboard accessibility milti hai. Agar sirf <div> use karo, to keyboard se navigate nahi ho sakta. Google bhi list-based navigation ko zyada samajhta hai.

05 Mobile Menu

☰ Hamburger Menu — Mobile Navigation

Mobile screen par sab links nahi aa sakti ek line mein. Hamburger icon (☰) click karne par menu slide karke khulta hai — CSS + JS mila ke banta hai.

HTML + CSS + JS — Hamburger Menu
/* ── HTML ── */
<!-- 3 spans = 3 lines of hamburger icon -->
<button class="hamburger" id="ham-btn"
        aria-expanded="false"
        ↑ Accessibility: open/close state
        aria-label="Navigation menu">
  <span class="line"></span>  <-- Top line
  <span class="line"></span>  <-- Middle line
  <span class="line"></span>  <-- Bottom line
</button>

<ul class="nav-links" id="nav-menu">
  ...links...
</ul>

/* ── CSS ── */
.hamburger .line {
  display: block;
  width: 25px; height: 2px;
  background: #333;
  border-radius: 2px;
  transition: all 0.3s ease;
  /* X animation ke liye */
}
/* Jab open ho — teen lines X ban jaati hain */
.hamburger.open .line:nth-child(1) {
  transform: translateY(7px) rotate(45deg);
}
.hamburger.open .line:nth-child(2) {
  opacity: 0; /* middle chhup jaati hai */
}
.hamburger.open .line:nth-child(3) {
  transform: translateY(-7px) rotate(-45deg);
}

/* Mobile menu — pehle chupaao */
@media(max-width: 768px) {
  .nav-links {
    display: none;
    position: absolute;
    top: 70px; left: 0;
    width: 100%;
    flex-direction: column;
    background: white;
    padding: 20px;
    box-shadow: 0 10px 30px rgba(0,0,0,0.1);
  }
  .nav-links.open {
    display: flex; /* JS se ye class add hogi */
  }
}

/* ── JAVASCRIPT ── */
const hamBtn = document.getElementById('ham-btn');
const navMenu = document.getElementById('nav-menu');

hamBtn.addEventListener('click', () => {

  // 'open' class add/remove karo
  hamBtn.classList.toggle('open');
  navMenu.classList.toggle('open');

  // Accessibility update
  const isOpen = navMenu.classList.contains('open');
  hamBtn.setAttribute('aria-expanded', isOpen);
});

// Link click par menu band karo
navMenu.querySelectorAll('a').forEach(link => {
  link.addEventListener('click', () => {
    hamBtn.classList.remove('open');
    navMenu.classList.remove('open');
  });
});

<section id="home">
Semantic Section + ID

id="home" isliye diya taaki navbar ka <a href="#home"> yahan jump kar sake. <section> Google ko batata hai ye ek alag content block hai.

<h1> — Sirf Ek!
Most Important SEO Rule

Poore page mein sirf EK <h1> hona chahiye. Google ise primary keyword maanta hai. Isme apna main keyword zaroor daalo (jaise: "Web Developer Mumbai").

clamp(2rem,4vw,3.5rem)
Fluid / Responsive Typography

min=2rem, preferred=4vw screen width ka, max=3.5rem. Screen size ke saath font automatically adjust hoga — media queries ki zarurat nahi for font size.

loading="eager"
Hero Image Fast Load

Hero image immediately load ho — warna user blank area dekhega. Baaki images "lazy" rakho — scroll par load honge, page fast rahega.

Table Ko 3 Parts Mein Divide Karo

Browser printing mein header/footer repeat karta hai. Screen readers bhi inhe alag samajhte hain. CSS styling bhi alag-alag ho sakti hai.

scope="col/row"
Accessibility Attribute

Screen readers ko batata hai: ye <th> column ka header hai ya row ka. Visually impaired users ke liye table padhna easy ho jaata hai.

colspan="2"
Cells Merge Karo

colspan se horizontal cells merge hoti hain. rowspan se vertical merge. Jaise Excel mein "Merge Cells" karte hain.

border-collapse: collapse
Double Border Hatao

Default mein har cell ki alag border hoti hai — double line dikhti hai. collapse se sab borders ek ho jaati hain — clean look milta hai.

const form = document.getElementById('contact-form');

form.addEventListener('submit', async (e) => {
  e.preventDefault();
  // Default submit rok

  let isValid = true;

  // Name validate karo
  const name = document.getElementById('name');
  const nameErr = document.getElementById('name-error');

  if (name.value.trim().length < 2) {
    name.classList.add('invalid');
    nameErr.classList.add('show');
    isValid = false;
  } else {
    name.classList.remove('invalid');
    nameErr.classList.remove('show');
  }

  if (!isValid) return;

  // Submit karo
  const btn = document.getElementById('submit-btn');
  btn.disabled = true;
  btn.textContent = 'Sending... ✈️';

  try {
    await fetch(form.action, {
      method: 'POST',
      body: new FormData(form),
    });
    btn.textContent = 'Bhej Diya! ✅';
    form.reset();
  } catch (err) {
    btn.textContent = 'Error! Try Again ❌';
    btn.disabled = false;
  }
});
